# 系统管理员接口文档

## 一、社团列表管理

### 1. 获取所有社团列表
- **接口名称**: 分页查询社团列表
- **URL**: `GET /club/list`
- **功能描述**: 查询所有社团列表,支持分页和关键词搜索

---

## 二、社团管理功能

### 2. 创建社团
- **接口名称**: 创建社团
- **URL**: `POST /admin/club/create`
- **功能描述**: 创建新的社团,需要管理员权限

### 3. 更新社团信息
- **接口名称**: 更新社团信息
- **URL**: `PUT /admin/club/update`
- **功能描述**: 修改社团基本信息,需要管理员权限

### 4. 删除社团
- **接口名称**: 删除社团
- **URL**: `DELETE /admin/club/{id}`
- **功能描述**: 删除社团及相关数据,需要管理员权限

### 5. 设置社团负责人
- **接口名称**: 设置社团负责人
- **URL**: `POST /admin/club/{clubId}/leader/{userId}`
- **功能描述**: 将指定用户设置为社团负责人,需要管理员权限

### 6. 取消社团负责人
- **接口名称**: 取消社团负责人
- **URL**: `DELETE /admin/club/{clubId}/leader/{userId}`
- **功能描述**: 取消指定用户的社团负责人身份,需要管理员权限

### 7. 审核社团加入申请
- **接口名称**: 审核社团申请
- **URL**: `POST /admin/club/applications/review`
- **功能描述**: 审核通过或拒绝用户的加入申请,需要管理员权限

### 8. 查询待审核申请列表
- **接口名称**: 查询待审核申请列表
- **URL**: `GET /admin/club/applications/pending`
- **功能描述**: 查看所有待审核的社团加入申请,需要管理员权限

---

## 三、社团成员管理功能

### 9. 移除社团成员 (新增)
- **接口名称**: 移除社团成员
- **URL**: `DELETE /admin/club/{clubId}/member/{userId}`
- **功能描述**: 将指定成员从社团中移除,需要管理员权限

### 10. 修改社团成员角色 (新增)
- **接口名称**: 修改社团成员角色
- **URL**: `PUT /admin/club/{clubId}/member/{userId}/role`
- **功能描述**: 修改成员在社团中的角色(member-普通成员, leader-负责人),需要管理员权限

---

## 四、活动管理功能

### 11. 查看所有活动
- **接口名称**: 查看所有活动
- **URL**: `GET /admin/activity/list`
- **功能描述**: 查看所有活动,支持关键词和状态筛选,需要管理员权限

### 12. 创建活动
- **接口名称**: 创建活动
- **URL**: `POST /admin/activity/create`
- **功能描述**: 系统管理员创建新活动,需要管理员权限

### 13. 更新活动
- **接口名称**: 更新活动
- **URL**: `PUT /admin/activity/{id}`
- **功能描述**: 系统管理员更新活动信息,需要管理员权限

### 14. 审核活动
- **接口名称**: 审核活动
- **URL**: `PUT /admin/activity/{id}/review`
- **功能描述**: 审核活动(通过/拒绝),需要管理员权限

### 15. 取消活动
- **接口名称**: 取消活动
- **URL**: `PUT /admin/activity/{id}/cancel`
- **功能描述**: 系统管理员取消活动,需要管理员权限

### 16. 删除活动
- **接口名称**: 删除活动
- **URL**: `DELETE /admin/activity/{id}`
- **功能描述**: 删除指定活动,需要管理员权限

### 17. 查看活动报名列表
- **接口名称**: 查看活动报名列表
- **URL**: `GET /admin/activity/{id}/signups`
- **功能描述**: 系统管理员查看任意活动的所有报名人员,支持分页,需要管理员权限

### 18. 活动签到/标记缺席
- **接口名称**: 活动签到/标记缺席
- **URL**: `POST /admin/activity/{id}/checkin`
- **功能描述**: 系统管理员为报名用户签到或标记缺席,需要管理员权限

---

## 说明

- 所有标注"需要管理员权限"的接口都需要用户具有 `system_admin` 角色
- 接口路径中的 `{id}`, `{clubId}`, `{userId}` 为路径参数,需要替换为实际的ID值
- 成员管理接口(第9、10项)补充了系统管理员对社团成员的完整管理能力
- 活动管理接口(第12-18项)补充了系统管理员对活动的完整生命周期管理能力,包括创建、更新、取消、审核、删除、查看报名和签到管理
